## 질문 목록 (8문항)

---

### Q1. 축 A: 안정 ↔ 실험

**팀에서 새 프레임워크 도입을 제안받았다. 당신의 반응은?**

- A) 기존 시스템과의 호환성부터 따진다 → A-안정
- B) 흥미롭다, 바로 PoC를 만들어본다 → A-실험

---

### Q2. 축 B: 구조 ↔ 즉흥

**새 프로젝트를 시작할 때 가장 먼저 하는 일은?**

- A) 아키텍처와 폴더 구조부터 설계한다 → B-구조
- B) 일단 코드부터 짜면서 방향을 잡는다 → B-즉흥

---

### Q3. 축 C: 개인 ↔ 협업

**프로덕션 버그가 발생했다. 당신의 첫 반응은?**

- A) 로그부터 파고들어 혼자 원인을 추적한다 → C-개인
- B) 관련 팀에 바로 공유하고 같이 해결한다 → C-협업

---

### Q4. 축 D: 품질 ↔ 속도

**배포 직전, 사소한 UI 버그를 발견했다. 어떻게 하겠는가?**

- A) 사소해도 고치고 배포한다 → D-품질
- B) 핫픽스로 다음에 처리하고 일단 배포한다 → D-속도

---

### Q5. 보조 구분: 설계 vs UX vs 부채청산

**가장 보람을 느끼는 순간은?**

- A) 깔끔한 아키텍처가 완성되었을 때 → 설계
- B) 사용자가 "이거 편하다"고 말할 때 → UX
- C) 지저분한 레거시 코드를 깔끔하게 정리했을 때 → 부채청산
- D) 밤새 디버깅 끝에 원인을 찾았을 때 → 몰입

---

### Q6. 보조 구분: 자동화 vs 연결 vs 몰입

**사이드 프로젝트를 한다면?**

- A) 개발 생산성을 높이는 자동화 도구 → 자동화
- B) 여러 API를 연결하는 매시업 서비스 → 연결
- C) 새로 나온 기술 프레임워크 실험 → (도깨비 보강)
- D) 하나의 문제에 깊이 파고드는 디버깅 도구 → 몰입

---

### Q7. 보조 구분: 테스트 vs 거버넌스

**코드 리뷰에서 가장 신경 쓰이는 부분은?**

- A) 엣지 케이스와 에러 처리가 충분한지 → 테스트
- B) 컨벤션과 팀 규칙을 잘 따르고 있는지 → 거버넌스
- C) 전체 설계 방향이 맞는지 → 설계
- D) 불필요한 복잡도는 없는지 → (속도 보강)

---

### Q8. 보조 구분: 최종 확인

**이상적인 개발 문화는?**

- A) 명확한 프로세스와 체계적 리뷰가 있는 문화 → 거버넌스
- B) 자유롭게 실험하고 빠르게 실패할 수 있는 문화 → (실험 보강)
- C) 각자 깊이 몰입할 수 있는 자율적 문화 → 몰입
- D) 자동화와 효율을 극대화하는 문화 → 자동화

## 점수 산출 로직

### 1단계: 4축 점수 (Q1~Q4)

각 질문에서 선택한 방향에 +1점:

```
A축: 안정(-1) / 실험(+1)
B축: 구조(-1) / 즉흥(+1)
C축: 개인(-1) / 협업(+1)
D축: 품질(-1) / 속도(+1)
```

4축 조합으로 1차 후보군 결정:

| 4축 조합 | 1차 후보군 |
|---|---|
| 안정, 구조, 개인, 품질 | 갓쓴키로 / 처녀귀신 / 저승사자 |
| 안정, 즉흥, 개인, 속도 | 총각귀신 |
| 실험, 즉흥, 개인, 속도 | 도깨비 |
| 실험, 구조, 개인, 속도 | 구미호 |
| 안정, 구조, 협업, 품질 | 해태 / 장승 |
| 안정, 즉흥, 협업, 속도 | 물귀신 |
| 그 외 조합 | 가장 유사한 프로필로 매칭 |

### 2단계: 보조 태그 (Q5~Q8)

Q5~Q8에서 획득한 보조 태그 점수로 1차 후보군 내 최종 캐릭터 결정:

```
후보군 [갓쓴키로, 처녀귀신, 저승사자]:
  설계 태그 최다 → 갓쓴키로
  UX 태그 최다 → 처녀귀신
  부채청산 태그 최다 → 저승사자
  동점 → 갓쓴키로 (기본값)

후보군 [해태, 장승]:
  테스트 태그 최다 → 해태
  거버넌스 태그 최다 → 장승
  동점 → 해태 (기본값)

단독 후보 (총각귀신, 도깨비, 구미호, 물귀신):
  보조 태그로 확신도 보강 (결과 페이지 표현에 활용)
```

### 3단계: 그 외 조합 처리 (폴백)

4축 조합이 정확히 매칭되지 않는 경우, 유클리드 거리 기반으로 가장 가까운 캐릭터 매칭:

```
distance = sqrt(
  (userA - charA)^2 +
  (userB - charB)^2 +
  (userC - charC)^2 +
  (userD - charD)^2
)
```

가장 거리가 짧은 캐릭터 후보군에서 보조 태그로 최종 결정.
