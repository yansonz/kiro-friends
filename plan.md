## 👑 갓쓴키로형 – 전략 설계자

### 성향
- 구조 중심
- 아키텍처 우선
- 장기적 사고

### 추천 Kiro 활용
- 설계 검토 요청
- 리스크 분석
- 확장성 평가

### 추천 Kiro 기능
- Spec: 요구사항 → 설계 → 구현 태스크를 단계별로 정리하여 아키텍처 설계를 체계화
- Steering: 프로젝트 전반의 설계 원칙과 아키텍처 가이드라인을 Steering 파일로 관리
- Autopilot 모드: 설계 문서 기반으로 Kiro가 자율적으로 구현 진행, 전략가는 리뷰에 집중
- Custom Subagents: 프론트엔드/백엔드/인프라 등 도메인별 전문 에이전트를 분리하여 아키텍처 관점에서 최적 구성
- Checkpointing: 설계 변경 시 이전 상태로 즉시 롤백, 다양한 아키텍처 방향을 안전하게 실험
- Powers (AWS CDK, Terraform): 인프라 설계 시 Powers로 CDK/Terraform 베스트 프랙티스 즉시 적용

### AI 협업 방식
- "5년 유지보수 관점에서 평가해줘"
- "구조적 리스크 정리해줘"

### ⚠️ 빠지기 쉬운 함정
- 설계만 하다 구현 지연
- 완벽주의로 시작 지연

- 테스트 코드 생성

### 추천 Kiro 기능
- Autopilot 모드: 몰입 중 흐름을 끊지 않고 Kiro가 자율적으로 코드 수정 및 디버깅 수행
- Hooks (agentStop): 작업 완료 시 자동으로 테스트 실행하여 밤샘 중 실수 방지
- MCP: 외부 도구(DB, 모니터링 등) 연동으로 컨텍스트 전환 없이 디버깅 집중
- Checkpointing: 밤샘 중 과감한 시도 후 문제 발생 시 원클릭 롤백으로 안전망 확보
- Kiro CLI: 터미널에서 바로 에이전트 호출, IDE 전환 없이 몰입 상태 유지
- Powers (Datadog, CloudWatch): 모니터링 Powers로 프로덕션 이슈를 IDE 안에서 바로 추적

### AI 협업 방식
- "에러 가능성 5가지"
- "리팩토링 우선순위"

### ⚠️ 빠지기 쉬운 함정

### 성향

- 자동화 지향
- 효율 극대화

### 추천 Kiro 활용
- 워크플로우 체인 구성
- 프롬프트 템플릿화

### 추천 Kiro 기능
- Hooks: 이벤트 기반 자동화 체인 구성 (파일 저장 → 린트 → 테스트 → 배포 준비)
- Steering: 프롬프트 패턴과 출력 형식을 Steering 파일로 템플릿화하여 재사용
- MCP: 다양한 외부 도구를 MCP로 연결하여 Kiro 안에서 통합 자동화 파이프라인 구축
- Custom Subagents: 용도별 전문 에이전트(코드 리뷰봇, 문서 생성봇 등)를 직접 설계하여 자동화 극대화
- Skills: 반복 사용하는 워크플로우를 Skill 패키지로 만들어 필요할 때만 로딩하는 효율적 컨텍스트 관리
- Powers 빌더: 자신만의 Power(MCP + Steering + Hooks 번들)를 만들어 팀이나 커뮤니티에 공유

### AI 협업 방식
- 출력 형식 명확히 정의
- 단계 분리 요청

# 5. 👥 팀 시너지 분석

## 시너지 조합

- 갓쓴키로 + 도깨비 → 전략 + 실험
- 해태 + 저승사자 → 안정적 구조 개선
- 구미호 + 총각귀신 → 자동화 + 실행력
- 장승 + 갓쓴키로 → 구조 + 질서

## 긴장 관계 조합

- 도깨비 vs 해태 → 실험 vs 안정
- 저승사자 vs 총각귀신 → 리팩토링 vs 몰입 개발
- 처녀귀신 vs 도깨비 → 완성도 vs 속도
